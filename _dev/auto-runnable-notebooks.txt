#   - ``.github/workflows/main.yml`` ``tutorials-execute`` CI job
#     (runs on every PR — keeps the published outputs honest)
#
# Categorization rules:
#
#   🟢  Auto-runnable  — needs only ``pip install quantum-metal[gui]``.
#                       Listed below. Re-executed on every PR.
#   🟡  External-gated — needs Ansys HFSS/Q3D, gmsh + Linux X libs, a
#                       real GDS file from a fab, etc. NEVER auto-run;
#                       cached outputs are sacred. Maintainer refreshes
#                       locally when the relevant tool changes.
#   🟠  Interactive    — depends on the Qt event loop (mouse clicks
#                       etc.). Programmatically un-runnable. Skip.
#
# To add a notebook to the auto-runnable set:
#
#   1. Confirm it imports nothing outside ``quantum-metal[gui]`` deps
#      (no ``pyaedt``, no ``gmsh``, no ``pyEPR``, no
#      ``ansys.aedt.core``).
#   2. Confirm ``QISKIT_METAL_HEADLESS=1 jupyter nbconvert --execute
#      <path> --to notebook`` completes in under 3 minutes.
#   3. Append the canonical ``docs/tut/`` path below. The CI executor
#      uses the docs/tut/ side; ``_dev/sync_two_folders.py`` will then
#      copy the refreshed outputs to ``tutorials/``.
#
# Lines starting with ``#`` are comments. Blank lines ignored.
