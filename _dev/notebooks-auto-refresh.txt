# Auto-refresh notebooks — outputs ARE overwritten in place by CI.
#
# Pure-Python / matplotlib-only notebooks that don't benefit from a
# full Qt window grab. ``_dev/rerun_auto.py`` runs these with
# ``--inplace``; the refreshed outputs land back in the source file
# and get committed by whoever's running the local refresh, OR end up
# in the diff CI produces.
#
# Categorization rules
# --------------------
#
# Add a notebook here when ALL of the following hold:
#
#   1. It needs only ``quantum-metal[gui]`` deps (no Ansys, no gmsh).
#   2. It doesn't ``gui.screenshot()`` the Qt window — its visuals come
#      from ``qm.view``, ``plt.subplots``, ``matplotlib.pyplot``, or
#      ``MetalGUIHeadless.screenshot()``. The published render is
#      semantically equivalent under headless and Qt execution.
#   3. It completes in under 3 min on a clean CI runner.
#
# If a notebook starts carrying Qt-specific screenshots (rare), move it
# to ``_dev/notebooks-frozen-qt.txt``.
#

# list: Ansys / gmsh / KLayout / Cross-Resonance notebooks that need a
# real toolchain installed and are NEVER auto-run. Outputs there are
# refreshed manually by the maintainer.
